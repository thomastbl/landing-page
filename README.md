## üõ†Ô∏è Development Workflow

To ensure efficient construction, a simple yet productive workflow was implemented.

### üî• Live Preview & Hot Reload

I used `npx live-server` to run a local development server with automatic refresh.

### üñ•Ô∏è Multi-Screen Setup

Coding was done on a Mac (VSCode) while displaying the final render on a large PC monitor, accessible via the Mac's local IP address.

This optimized:
* Accurate visual feedback on large desktop screens.
* Adjustment of spacing, alignment, and Hero proportions.
* Verification of Flexbox behavior on wide screens.

### üß© Debugging and Layout Tools

Temporary CSS border utilities were used to visualize the boxes.

### üì± Responsive Behavior Testing

Tests were performed using the Chrome DevTools mobile emulator (`Cmd + Shift + M`) and manual resizing, which allowed for the validation of:
* The effect of the `meta viewport` on mobile screens.
* How Flexbox automatically stacks or wraps elements.
